leyu · 从一处角落开始

一个关于清晰、安静与连贯体验的叙述。不是所有页面都需要喧嚣，leyu 更倾向用克制的结构承载内容。

几个自然的理由

结构优先

标题、段落、留白——每一层都有存在的理由。不堆砌，不冗余，阅读节奏由内容自己决定。

克制的美学

低对比底色、窄栏、轻量阴影。视觉让位于文字，界面退后半步，信息自然浮现。

跨屏的连贯

从手机到桌面，网格与间距自动适配。不需要特意缩放，每一段都保持呼吸感。

叙述的语气

不承诺奇迹，不制造焦虑。用平实的句子解释功能、场景与边界，像一次坦诚的对话。

leyu 背后的主题

关键词从来不只是标签。当提到 leyu，它指向一种对网页底色的思考：
一个页面应该怎样被阅读？答案或许不在于添加更多元素，而在于筛选、排序、然后沉默地呈现。

这里的每个区块都经过审视——hero 不需要大图，优势不需要图标，FAQ 不需要折叠动画。信任建立在内容的真实排列上，而不是特效的堆叠。这是一种安静的主张。

你可以把 leyu 看作一个出发点。它提供一种基准：干净的排版、克制的对比、不过度承诺的文案。从这里开始构建任何面向内容的页面，都会自带一种沉稳的基调。

对于希望页面回归阅读本质的人来说，leyu 是一个低干预的框架。不定义你的内容，只是为内容提供一个恰当的容器。

一些人们常问的事

leyu 适合什么样的网站？
个人页面、文档、小型项目介绍、轻量品牌站——任何需要以文字为主要表达方式的地方。如果页面需要大量图片或复杂交互，leyu 也可以作为基础层，再灵活叠加。
会不会缺少常见的组件？
有意保持精简。导航、按钮、卡片、表单等要素都可用简单 HTML 与少量 CSS 扩展。leyu 提供视觉基调，而不是组件库。
配色和字体可以自定义吗？
当然。style 区域中的变量与选择器都采用极简方式，替换颜色值、字族或间距即可重塑气质。leyu 不锁定任何样式。
